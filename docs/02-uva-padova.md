# UVA/Padova T1D Simulator: что нужно знать

UVA/Padova S2013 — единственный симулятор diabetes type 1, который **прошёл FDA acceptance** как замена доклиническим испытаниям на животных (2008, обновлён 2013). На его основе валидируются Medtronic SmartGuard, Tandem Control-IQ, и все open-source петли.

Для нас он важен как:
1. Источник синтетических данных для предобучения.
2. Forward-model в обратной задаче: f(C, профиль) → G(t).
3. Эталон, относительно которого валидируем свой ODE-solver.

## Физическая модель в одном экране

13 ODE, описывают четыре подсистемы:

### 1. Желудочно-кишечный тракт (gut absorption)
```
Q_sto1' = -k_gri · Q_sto1 + D · δ(t - t_meal)     [твёрдая фаза в желудке]
Q_sto2' = k_gri · Q_sto1 - k_empt(Q_sto) · Q_sto2  [жидкая фаза]
Q_gut'  = k_empt · Q_sto2 - k_abs · Q_gut          [тонкий кишечник]
Ra(t)   = f · k_abs · Q_gut / BW                   [скорость поступления в кровь]
```
- `D` — съеденные углеводы (г) — **то, что мы хотим восстановить**.
- `k_empt` — скорость опорожнения желудка — нелинейная функция, замедляется при больших порциях. Зависит от типа еды (жир, белок) — **то, что мы хотим восстановить как τ_meal**.

### 2. Глюкозная подсистема
```
G_p'  = EGP(t) + Ra(t) - U_ii(t) - E(t) - k_1 · G_p + k_2 · G_t
G_t'  = -U_id(t) + k_1 · G_p - k_2 · G_t
```
- `EGP` — эндогенная продукция глюкозы печенью (подавляется инсулином).
- `U_ii, U_id` — insulin-independent / insulin-dependent утилизация.
- `E` — почечная экскреция (при сильной гипергликемии).

### 3. Инсулиновая подсистема
```
I_l' = -(m_1 + m_3) · I_l + m_2 · I_p + R_a^I(t)   [печень]
I_p' = -m_4 · I_p + m_1 · I_l - m_2 · I_p           [плазма]
X'   = -p_2U · X + p_2U · (I - I_b)                  [компартмент действия]
```
- `R_a^I` — поступление подкожного инсулина в кровь (своя 2-компартмент подсистема).
- `X` — «эффективный» инсулин, действующий на ткани.

### 4. Подсистема измерения (CGM)
```
G_M' = -(1/τ_CGM) · (G_M - G_p)
```
CGM измеряет интерстициальную жидкость с задержкой τ_CGM ≈ 6–10 минут и шумом ~10%. **Это критично:** мы наблюдаем не G_p, а G_M, и должны деконволвить.

## Параметры

В модели ~40 параметров. Половина — фиксированные физиологические константы. Половина — индивидуальные:
- `BW` — масса тела.
- `V_g, V_i` — объёмы распределения.
- `k_p1..k_p4` — параметры EGP.
- `k_abs, f` — абсорбция углеводов.
- `k_a1, k_a2, k_d` — фармакокинетика инсулина (пик ~75 мин, DIA ~5 ч).
- ...

В FDA-сертифицированной версии есть **30 виртуальных пациентов** с зафиксированными параметрами (10 adult / 10 adolescent / 10 child). Это наш test-set для синтетических экспериментов.

## Что важно для нашей задачи

**Хорошие новости:**
1. Параметры из истории пациента можно восстановить (это делает AAPS Autotune). Значит, индивидуализация возможна.
2. Прямая модель быстрая (~ms на 24-часовую симуляцию на CPU).
3. Дифференцируемая (можно прогнать gradient через ODE-solver) → подходит для PINN.

**Плохие новости:**
1. Параметров много, обратная задача недоопределена.
2. Реальная физиология отклоняется от модели (стресс, болезнь, цикл, медикаменты).
3. CGM-шум маскирует тонкие сигналы.

## Что мы реализуем сами

Полная UVA/Padova S2013 — overkill для нашей задачи. Мы возьмём **упрощённую 4-уравнениевую версию** (минимальная модель Бергмана + meal compartment), достаточную для:
- forward simulation за <1 ms,
- дифференцирования по C, τ,
- запуска на мобильнике через ONNX.

Полная модель используется только как **источник синтетических данных** для предобучения.
