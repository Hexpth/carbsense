# Benchmark: CarbSense MAP estimator vs SimGlucose

**Purpose.** Quantify how well our simplified Bergman-based forward model and
the MAP inverse solver recover meal carbohydrate amounts from CGM traces
generated by the FDA-validated UVA/Padova S2013 simulator (SimGlucose).

This is the first *independent* check on the inverse pipeline: SimGlucose
uses a different (full, non-simplified) forward model, so any model
mismatch between CarbSense and physiological reality is exposed here.

## How to reproduce

```bash
pip install -e ".[sim]"
python scripts/run_simglucose_benchmark.py
```

This runs 10 adult patients × 4 meal sizes (30, 50, 75, 100 g) = **40 episodes**.
Wall time on a laptop CPU: ~5–10 minutes.

A per-episode CSV is written to `benchmark_simglucose.csv` and a summary table
is printed to stdout.

## Interpretation rules (decided up front)

| Metric on full benchmark | Verdict | Action |
|---|---|---|
| MAE < 15 g | Simplified forward model is sufficient. | Proceed to neural amortizer (Phase M10+ of roadmap). |
| 15 g ≤ MAE < 25 g | Marginal. Acceptable for a baseline, but limits the ceiling for the final paper. | Consider extending to full UVA/Padova S2013 in `physiology/uvapadova_full.py`. |
| MAE ≥ 25 g | Significant model mismatch. | Debug the forward operator before adding any ML. |

These thresholds were chosen *before* seeing the numbers — to keep the
verdict honest. State-of-the-art on real free-living data is ~18–25 g
[Mahmoudi 2019, Zheng 2022], so anything in the < 25 g zone on synthetic
matched-noise data is plausible.

## Caveats / known limitations of this benchmark

1. **Single-meal scenarios only.** No snacks, no overlapping events. This is
   the easiest case. Real data will be harder.

2. **Basal insulin only.** No correction boluses, no meal boluses. This
   isolates the carb-rise signal but is unrealistic for free-living T1D.
   Adding insulin confounders is a follow-up benchmark.

3. **Default `PatientParams`.** We do not personalise the inverse model to
   each SimGlucose patient. A future experiment should run Autotune-style
   parameter fitting first and then re-evaluate MAE.

4. **Adult cohort only.** Adolescent and child SimGlucose patients have
   markedly different insulin sensitivity and gastric emptying. Expected to
   be a future benchmark axis.

5. **Warm-started `t_meal_init`.** We seed the optimizer with the true
   `t_meal − 10 min`. In production the meal time would not be known.
   Detecting the onset is a separate problem (see Dassau 2008 baseline).
